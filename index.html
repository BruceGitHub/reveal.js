Docker in sviluppo

  • Usato nel 2014
  • Cross OS
  • Niente più ingombranti VM

Obiettivi di Docker in prod

  • Stesso ambiente dev-prod
  • Uso "vero" del cloud (K8S)
  • App realmente pacchettizzata
  • Solo le app, non i servizi con persistenza!
  • CD, perché si!

Dockerfile

  • File base + ci/dev/prod
  • Base Alpine
  • Installare tutto in maniera pulita
  • Lock di tutte le versioni
  • No root!
  • Utente base 1000 che funzioni anche in dev linux
  • Composer (perché tanto serve dopo)

Dockerfile dev

  • xDebug
  • Mount volumi
  • Node? Container a parte!

Dockerfile prod

  • Cache-friendly
  • Prima composer, poi install no script, poi src, poi install
  • Stesso dockerfile, Symfony env in ARG
