Le regole

  1. Use only one level of indentation
  2. Don’t use the else keyword
  3. Wrap all primitives and strings (not always)
  4. Use only one dot -> per line
  5. Do NOT abbreviate
  6. Keep your classes small
  7. Don’use any classes with more than
    two to five instance variables
  8. Use first-class collections
  9. Don’t use accessors (getters/setters)
  10. Document your code
